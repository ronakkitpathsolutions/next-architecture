# Antigravity Agent Instructions

## Editor & Formatting Configuration

This project uses the following editor and formatting settings. **All code generated by the agent MUST comply with these rules.**

### EditorConfig Rules (`.editorconfig`)

| Setting                | Value |
| ---------------------- | ----- |
| `indent_style`         | space |
| `indent_size`          | 2     |
| `end_of_line`          | lf    |
| `charset`              | utf-8 |
| `insert_final_newline` | true  |

### VS Code Settings (`.vscode/settings.json`)

| Setting                   | Value                        |
| ------------------------- | ---------------------------- |
| `editor.formatOnSave`     | true                         |
| `editor.defaultFormatter` | prettier                     |
| `prettier.requireConfig`  | true                         |
| `i18n-ally.localesPaths`  | `public/locales`, `src/i18n` |
| `cSpell.words`            | `unrs`                       |

### Prettier

- Prettier is the project formatter (integrated via `eslint-plugin-prettier`)
- `prettier.requireConfig` is enabled — Prettier will only format if a config is found
- Formatting scripts: `bun run format` (write) and `bun run format:check` (check)

### ESLint

- ESLint flat config is used (`eslint.config.js`)
- Includes: `eslint-plugin-prettier/recommended`, `eslint-config-next`, TypeScript-ESLint, React Hooks, React Refresh
- Lint scripts: `bun run lint` and `bun run lint:fix`

## Code Style Rules (Summary)

When generating or modifying code, always:

1. **Use 2-space indentation** (no tabs)
2. **Use LF line endings** (not CRLF)
3. **Include a final newline** at the end of every file
4. **Use UTF-8 encoding**
5. **Follow kebab-case** for file names
6. **Follow PascalCase** for components, **camelCase** for hooks/functions/variables, **UPPER_SNAKE_CASE** for constants
7. **Use TypeScript strict mode** — `any` is forbidden
8. **Use Tailwind CSS** for styling (preferred)
9. **Package manager**: Bun (use `bun run` instead of `npm run`)
